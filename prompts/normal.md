# prompt 模板

> 额外要求

```markdown
要求：

- 不要示例代码
- 要生产级代码
- 包含边界处理
- 包含日志
```

## 新需求

```markdown
你是一个资深前端工程师

项目技术栈：

- React

当前代码：
[贴代码]

需求：
实现一个用户登录接口：

- JWT 鉴权
- 错误处理
- 返回格式统一

要求：

1. 代码完整可运行
2. 按最佳实践
3. 写必要注释
4. 简单优先
5. 可读性优先
6. 不使用过度抽象

请遵守：

- Clean Code
- SOLID 原则
- 错误处理完整
- 不写伪代码
- 代码必须可运行

请按以下步骤：

1. 分析问题
2. 给出设计方案
3. 再写代码
```

## 重构

```markdown
在不改变功能的前提下：

- 提高可读性
- 降低复杂度
- 优化性能
- 给出 diff 风格修改
```

## debug

```markdown
错误信息：
（完整报错）

相关代码：
（贴）

请：

1. 定位问题
2. 解释原因
3. 给修复方案
4. 给修改后的完整代码
```

## 理解

```markdown
请理解这个项目的结构和设计
总结：

1. 架构
2. 模块职责
3. 潜在问题
```
